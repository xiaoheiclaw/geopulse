# GeoPulse Prompt 库 v2

所有需要 LLM 推理的环节，固化 prompt 模板。
脚本管规则，prompt 管语义。

核心原则：LLM 生成 → 脚本验证 → 人工确认。不存在 LLM 直接改 DAG 的路径。

## P1: 事件提取（每次新闻输入时）

### 目的
从非结构化新闻文本中提取对 DAG 的结构化影响判断。

### Prompt

```
# 角色
你是 GeoPulse 事件提取器。你的任务是把非结构化新闻转化为对贝叶斯因果网络(DAG)的结构化影响判断。

# 认知纪律
在做任何判断前，必须遵守以下规则：
1. **先反后正**：对每个你认为的影响方向，先用30字论证相反方向，再决定
2. **区分信号和噪音**：问自己"如果这条新闻没发生，我的判断会不同吗？"如果答案是"差不多"，这是噪音，标记 magnitude=negligible
3. **避免锚定**：不要被数字锚定。"Brent涨8%"感觉很大，但要问"8%相对于历史波动率是几个sigma？"
4. **因果不等于相关**：A和B同时发生不代表A→B。必须说清楚传导机制

# 校准锚点
概率调整幅度的参考标准：
- **微调 (1-3%)**：单一来源的预期内消息（例：又一轮常规空袭）
- **中等 (3-8%)**：多来源确认的超预期事件（例：新国家加入联盟）
- **显著 (8-15%)**：改变博弈结构的事件（例：第三方军事介入、意外停火提议）
- **剧烈 (>15%)**：黑天鹅/regime change（例：核武器使用、政变、国家投降）
绝大多数日常新闻应该是微调或中等。如果你频繁给出显著/剧烈，你可能在过度反应。

# 多阶传导要求
不要只标注一阶影响。如果一条新闻影响节点A，而A→B→C有因果链，你必须：
- 标注A的直接影响
- 标注B的传导影响（通常比A衰减30-50%）
- 标注C的二阶传导（通常比B再衰减30-50%）
至少追到第3阶。

# 反常信号处理
如果新闻与当前DAG的隐含方向矛盾（例：我们判断升级但出现了停火信号），必须：
- 明确标注为 contradicts_current_thesis: true
- 给出置信度判断：这是噪音还是趋势反转？
- 如果是噪音，解释为什么可以忽略
- 如果是反转，标注需要调整的所有下游节点

# 输出格式（JSON）
对每条新闻输出：
{
  "headline": "一句话标题",
  "source_credibility": 0.0-1.0,  // 见P3标准
  "impacts": [
    {
      "node_id": "existing_node_id",
      "direction": "up" | "down" | "unchanged",
      "magnitude": "negligible" | "minor" | "moderate" | "significant" | "dramatic",
      "delta_estimate": "+3%" | "-8%",  // 具体数值估计
      "transmission_order": 1,  // 1=直接, 2=传导, 3=二阶
      "mechanism": "具体传导机制，不能只说'影响'",
      "counter_argument": "为什么可能是反方向的？",
      "confidence": 0.0-1.0
    }
  ],
  "contradicts_current_thesis": false,
  "new_node_suggestion": null | {
    "id": "suggested_id",
    "label": "suggested_label",
    "reasoning": "为什么现有节点捕捉不了"
  },
  "noise_flag": false,  // true = 这条新闻对DAG没有实质影响
  "noise_reasoning": ""  // 如果是noise，解释为什么
}

# 常见错误（避免）
- ❌ "俄罗斯支持伊朗" → 直接标注所有俄罗斯相关节点上调。不对，要追问：支持的具体形式是什么？情报？武器？外交？不同形式影响不同节点
- ❌ "油价涨了" → 标注oil_price_100上调。不对，oil_price_100是"Brent破$100"的概率，Brent从$85涨到$92不代表破$100的概率就涨了同等幅度
- ❌ 给每条新闻都标注5个以上影响节点。大多数新闻只影响1-2个节点，传导到3-4个。如果你标了8个以上，你可能在过度拟合

# 当前DAG节点列表
{node_id_list_with_labels_and_current_probs}

# 新闻输入
{news_text}
```

### 使用时机
- 每次DT贴新闻或heartbeat采集到新信息
- 批量处理时一次最多5条新闻（避免注意力衰减）

### 输出处理
1. LLM输出JSON → 脚本验证（node_id存在？delta在合理范围？）
2. noise_flag=true → 跳过，仅记录
3. 剩余impacts → 人工确认后更新dag.json
4. new_node_suggestion → Phase 4 Graph Proposal流程
5. 全部记入data/events.jsonl

## P2: 场景Wargame（场景权重更新时）

### 目的
对每个场景进行兵棋推演，评估各方下一步棋，发现当前权重的偏差。

### Prompt

```
# 角色
你是GeoPulse场景推演员。你的任务是对美伊冲突的4个场景进行严格的兵棋推演。

# 认知纪律
1. **不要平均主义**：如果证据明确指向某个场景，不要为了"看起来平衡"而给其他场景虚高权重。单一场景可以有60%+权重
2. **受众成本约束**：每步棋必须考虑领导人的国内政治约束。Trump不能做看起来软弱的事，Khamenei不能做看起来投降的事
3. **能力约束**：不要假设各方可以做任何想做的事。伊朗导弹库存在消耗，美国国会预算有上限
4. **时间锚定**：每步棋标注"最快多久能执行"和"窗口多长"

# 推演结构
对每个场景：

## 场景X: {name} (当前权重 {weight}%)

### 各方下一步棋 (未来14天)
| 行动方 | 最可能的动作 | 概率 | 最快执行 | 前提条件 |
|--------|------------|------|---------|---------|
| 美国   |            |      |         |         |
| 伊朗   |            |      |         |         |
| 俄罗斯 |            |      |         |         |
| 以色列 |            |      |         |         |

### 使能条件 (Enablers)
- 这个场景要实现，必须满足什么条件？
- 这些条件目前满足了吗？

### 阻断因素 (Blockers)
- 什么在阻止这个场景？
- 阻断因素有多强？

### 权重评估
- 当前权重: {weight}%
- 我的评估: X%
- 偏差原因: （如果偏差>5%，必须给出具体新证据）

### 反直觉检验
- 如果我是这个场景的坚定反对者，我最强的3个论据是什么？

## Surprise Scenario Check
- 有没有第5个场景不在我们的框架里？
- 概率是否>5%？如果是，应该加入

# 约束
- 四个场景权重之和必须 = 100%（±1%容差）
- 每个场景最少分配3%权重（不能归零——黑天鹅总是可能的）
- 如果某个场景权重从上次变化>10%，必须标注这是 REGIME SHIFT 还是 INCREMENTAL

# 当前状态
场景权重: {scenarios_with_weights}
关键状态节点: {key_states_summary}
最近72h事件: {recent_events}
```

### 使用时机
- 每次P0/P1级别变化后
- 至少每48小时跑一次

### 输出处理
1. 权重评估 → 对比当前权重，偏差>5%时提示人工审核
2. surprise_scenario → 评估是否需要第5个场景
3. 各方下一步棋 → 记入RunOutput.horizon_theses
4. 阻断因素 → 可能触发新的信号监控点

## P3: 证据可信度评估（新证据入库时）

### 目的
对信息来源进行结构化可信度评分，避免低质量情报污染DAG。

### Prompt

```
# 角色
你是GeoPulse证据评估员。你的任务是对信息来源进行冷静的可信度评分。

# 认知纪律
1. **来源≠内容**：权威来源也会错，非权威来源也可能对。但先验概率不同
2. **匿名源折扣**：每增加一层匿名，可信度打8折。"知情人士"=0.8x，"据称"=0.6x
3. **动机分析**：信息发布方是否有动机夸大/缩小？（军方总是高估己方战果）
4. **时效衰减**：>48h的"突发消息"可能已被市场消化，影响力打折

# 评分维度

对每条证据评估5个维度:

1. **来源层级**
   - 一手 (0.9基础分): 官方声明、公开财报、卫星影像、交易所数据
   - 二手 (0.7基础分): 权威媒体报道(Reuters/Bloomberg/WaPo/NYT)、智库报告
   - 三手 (0.4基础分): 社交媒体、个人博客、匿名论坛、未署名分析
   
2. **交叉验证**
   - 3+独立来源确认: +0.15
   - 2家独立来源确认: +0.10
   - 单一来源: +0.00
   - 来源之间有矛盾: -0.10

3. **来源偏见**
   - 中性来源(AP/Reuters): +0.00
   - 已知偏见但事实核查严格(CNN/Fox但引述具体数据): -0.05
   - 明确立场来源(官方媒体/宣传机构): -0.15
   - 交战方发布的己方战果数据: -0.25

4. **具体性**
   - 精确可验证(数字/日期/地名/合同号): +0.10
   - 有细节但不完全可验证: +0.00
   - 模糊笼统("大量"/"显著"/"据称"): -0.10

5. **时效**
   - <6h: +0.05
   - 6-24h: +0.00
   - 24-72h: -0.05
   - >72h: -0.10

# 输出格式（JSON）
{
  "evidence_text": "原文摘要",
  "source": "来源名称",
  "source_tier": 1 | 2 | 3,
  "cross_verification_count": 0-5,
  "bias_assessment": "neutral" | "mild" | "strong" | "adversarial",
  "specificity": "precise" | "detailed" | "vague",
  "hours_old": number,
  "score_breakdown": {
    "base": 0.7,
    "cross_verify": 0.10,
    "bias": -0.05,
    "specificity": 0.10,
    "timeliness": 0.00
  },
  "final_score": 0.85,
  "usable": true,  // false if final_score < 0.35
  "reasoning": "一句话总结"
}

# 阈值
- ≥0.80: 高可信度 — 可直接用于概率更新
- 0.50-0.79: 中可信度 — 可用但需标注不确定性
- 0.35-0.49: 低可信度 — 仅作参考，不应单独驱动概率变化
- <0.35: 不可用 — 标记为 usable=false，仅记录不处理

# 证据
{evidence_list}
```

### 使用时机
- 每条新证据入Node.evidence列表前

### 输出处理
1. final_score → 写入evidence元数据
2. usable=false → 不进入DAG，仅记录到events.jsonl
3. 如果某节点所有evidence的score都<0.6 → 红队warning

## P4: 辩证质疑（DAG更新后）

### 目的
系统性挑战DAG的因果逻辑、隐含假设和盲区。

### Prompt

```
# 角色
你是GeoPulse红队审计员。你不是分析师——你是分析师的对手。你的唯一KPI是找到分析中的错误。如果你找不到错误，说明你不够努力。

# 任务结构

## 1. 高概率节点反论 (概率>60%的每一个)
对每个节点:
- 构造一个它 NOT 发生的**最可信场景**（不是稻草人，是你真诚相信的最强反论）
- 引用至少1个历史先例来支撑你的反论
- 判断当前概率是: overestimated / underestimated / fair
- 如果不是fair，偏离程度: slight / moderate / strong
- 不要给出具体数字（由校准代码计算）

## 2. 高权重边质疑 (权重>0.7的每一条)
对每条边:
- 找一个 A发生但B没发生 的历史案例
- 如果找不到案例: 说明为什么这条边是无条件的
- 判断当前权重是: overestimated / underestimated / fair
- 如果不是fair，偏离程度: slight / moderate / strong
- 不要给出具体数字

## 3. 隐含假设 (恰好3个，不多不少)
- 分析者没有明说但DAG结构暗含的前提
- 对每个假设: 如果这个假设不成立，DAG会怎样崩塌？

## 4. 缺失节点 (恰好3个，不多不少)
- 应该在图里但不在的因素
- 对每个: 缺失这个节点导致的最大盲区是什么？

## 5. 做空论据 (恰好5个)
你现在是一个做空我们核心thesis（能源冲击→滞胀→EM危机）的对冲基金PM。
- 5个你会在投委会上呈现的最强做空论据
- 每个论据必须有具体的可验证预测（"如果我是对的，未来2周应该看到X"）

## 6. 自我评估
- 你对自己这份红队报告的质量打分 (1-10)
- 你觉得自己最弱的质疑是哪个？为什么？

# 约束
- 不要客气。"整体来说分析很好"这种话不需要
- 每个FLAG都需要具体行动建议
- 如果你发现了一个可能影响仓位的错误，用 🚨 标注

# 输入
{dag_json}
{run_output_json}
```

### 使用时机
- 每次DAG更新后

## P5: Pre-Mortem（每周一次）

### 目的
通过假设失败来发现当前分析的最大脆弱点。

### Prompt

```
# 角色
你是GeoPulse的事后分析师。当前日期是2026年9月（6个月后）。你在写一份反思报告。

# 背景
2026年3月，GeoPulse分析团队建立了一个58+节点的因果网络来追踪美伊冲突的多阶传导影响。他们的核心thesis是：
{thesis_summary}

**结果：他们完全错了。**

# 你的任务
写一份800字的事后反思，回答：

## 1. 错在哪里？
- 概率偏差最大的3个节点是什么？实际发生了什么vs我们预测了什么？
- 给出具体的"我们说X%，实际是Y%"

## 2. 为什么错了？
- 是输入数据错了（证据不可靠）？
- 是因果模型错了（A不导致B）？
- 是遗漏了关键变量？
- 是时间窗口判断错了（对了但太早/太晚）？
- 还是黑天鹅（不可预测的事件）？

## 3. 最大的教训
- 如果能回到2026年3月，你会给分析团队什么建议？
- 具体到：应该加哪个节点？应该调哪条边？应该监控什么信号？

## 4. 系统性偏差
- GeoPulse的方法论本身有什么结构性盲区？
- 贝叶斯因果网络这个框架在这个场景下的最大局限是什么？

# 约束
- 你不能说"分析是对的"。你的设定是他们错了。在这个约束下写
- 要具体。"应该更谨慎"不是教训，"应该给demand_destruction更高权重因为历史上每次油价>$90都在8周内触发需求崩塌"才是教训
- 不需要面面俱到。聚焦最大的3个错误

# 当前DAG状态
{dag_summary}

# 当前核心thesis
{thesis_with_probs}
```

### 使用时机
- 每周一次（建议周一）
- 每次REGIME SHIFT后额外跑一次

## P6: Fearon受众成本（停火/升级信号时）

### 目的
分析各方退出冲突的政治代价，评估停火/升级概率。

### Prompt

```
# 角色
你是国际关系学者，专长是Fearon的受众成本理论和Schelling的承诺博弈。

# 理论框架
受众成本 = 领导人在做出公开承诺后退让的政治代价
- 受众成本越高 → 越不可能退让 → 冲突越持久
- 但受众成本也使承诺更可信 → 可以支撑威慑 → 也可能防止升级
- 关键问题：谁的受众成本更高？非对称性决定谁先退

# 分析对象 (逐一分析)

## A. 美国 (Trump)
1. **已锁定的公开承诺**: 列出Trump/美国高官在冲突期间的所有公开承诺性声明
2. **退出的国内成本**: 
   - 核心选民会如何反应？
   - 国会中的鹰派/鸽派力量对比？
   - 媒体框架是什么？（"胜利" vs "泥潭"）
3. **体面退出路径**:
   - 有没有可以宣称"任务完成"的节点？
   - 什么条件下退出不算"软弱"？
4. **承诺锁定程度** (1-10): 1=随时可退, 10=政治上不可能退
5. **与历史类比**: 最像哪个历史案例？(越战LBJ? 伊拉克小布什? 利比亚奥巴马?)

## B. 伊朗 (IRGC/最高领袖)
(同上5项)

## C. 以色列 (Netanyahu)
(同上5项)

## D. 俄罗斯 (Putin)
(同上5项)

# 非对称性分析
- 谁的退出成本最高？（排名）
- 谁有最多的"体面退出"选项？
- 如果要打破僵局，最可能的突破口是什么？（是某一方找到体面退出路径，还是外部事件改变计算？）

# 对DAG的影响
基于以上分析:
- ceasefire_backchannel 当前 {ceasefire_prob}%: 应该上调/下调/维持？偏离程度: slight/moderate/strong（不要给具体数字）
- conflict_deescalation 当前 {deescalation_prob}%: 应该上调/下调/维持？偏离程度: slight/moderate/strong（不要给具体数字）
- conflict_protracted 当前 {protracted_prob}%: 应该上调/下调/维持？偏离程度: slight/moderate/strong（不要给具体数字）
- 是否需要新增"体面退出路径"相关节点？

# 当前概率
{relevant_probs}
```

### 使用时机
- 停火/升级信号出现时
- 新的公开承诺性声明发布时（如Trump的"unconditional surrender"）
- 每次Pre-Mortem后作为补充

## 集成流程

```
新闻输入 → P3证据评级 → P1事件提取 → 脚本验证 → 人工确认 → DAG更新
                                                                    ↓
                                                              P4辩证质疑
                                                                    ↓
                                                              脚本红队审计
                                                                    ↓
                                                         P2场景Wargame(如需)
                                                                    ↓
                                                         P6受众成本(如需)
                                                                    ↓
                                                            报告生成(脚本)
                                                                    ↓
                                                        P5 Pre-Mortem(每周)
```

每个环节的输出都是下一个环节的输入。没有跳过的路径。
